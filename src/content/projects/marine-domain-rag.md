## 문제

해양 법령 질문은 한 조문만 읽어서는 답이 나오지 않는 경우가 많습니다. "선박에서 기름이 유출되면 어느 법 어느 조문으로 처벌되는가" 같은 질문은 의무 조문, 벌칙 조문, 시행령의 위임 조문을 가로질러야 합니다. 임베딩 유사도만으로 검색하면 질문과 표현이 비슷한 조문은 찾아도, 참조 사슬로 연결된 조문은 놓칩니다. 조문 간 인용·준용·위임 관계를 검색에 반영하고, 답변마다 근거 조문을 정확히 인용하는 파이프라인이 목표였습니다.

## 실무 맥락과 공개 범위

슈어소프트테크 인턴십 중 해양경찰청 CDX 과제로 진행한 작업입니다. 과제에서는 도메인 특화 RAG DB 구축과 검색·답변 흐름 구성에 참여했고, 배포 전 단계까지 진행한 상태로 인턴십이 끝났습니다. 회사 보안상 사내 코드·데이터는 공개할 수 없어, 같은 문제를 공개 데이터로 처음부터 다시 만들어 검증한 것이 이 레포입니다. 평가 하네스와 웹 데모는 과제 이후 개인적으로 디벨롭한 부분입니다.

## 데이터: 법령 105개, 조문 7,506건

국가법령정보센터 OPEN API로 해양 법령 34개를 법률·시행령·시행규칙 세트로 수집해 문서 105개를 만들었습니다. 검색 단위는 조문입니다. 조문번호·가지번호(제N조의M)를 보존하고 항·호·목을 평탄화해 조문 7,506건(별표·서식 1,688건 포함)으로 색인했습니다. 법령은 개정되므로 공포번호를 비교해 바뀐 법령만 다시 수집하고, 해당 법령 조문만 삭제 후 업서트하는 증분 재색인 스크립트를 두었습니다.

## 검색: 세 전략과 RRF 병합

BM25(Kiwi 형태소 분석), 벡터(Qdrant + KURE-v1), 그래프(Neo4j) 세 경로를 만들고 RRF(k=60)로 병합합니다. 그래프 전략은 벡터·BM25 상위 후보를 시드로 삼아 인용 관계를 1홉 확장하고, 확장 결과를 세 번째 랭킹으로 RRF에 가산합니다. 텍스트 유사도가 약해도 참조 관계로 연결된 조문이 후보에 올라올 수 있게 한 설계입니다. 질문이 "「해양환경관리법」 제63조"처럼 조문을 직접 지목하면 정규식으로 파싱해 해당 조문을 1순위에 고정합니다. cross-encoder 리랭커(bge-reranker-v2-m3)는 선택 기능으로 두었습니다.

## 에이전트: ReAct 루프 대신 상태 그래프

오케스트레이션은 LangGraph StateGraph입니다. 분류 → 검색 → 근거 판정 → 생성 → 검증의 노드 7개를 조건부 엣지로 잇고, 근거가 부족하면 재작성된 질의로 재검색(최대 2회), 인용 검증에 실패하면 재생성(최대 2회)하며, 답할 수 없는 질문은 두 경로에서 거절합니다. LLM이 툴을 고르는 ReAct 루프가 아니라 검색 전략을 결정적 라우팅으로 선택하는 구조를 택했는데, 그 이유와 트레이드오프는 별도 글로 정리했습니다.

## 평가: 골드셋 180문항과 뒤집힌 가설

골드셋 180문항(멀티홉 117, 단일 29, 정의 8, 거절 26)을 만들어 검색 전략을 비교했습니다. 멀티홉 정답 체인 후보는 그래프에서 구조적으로 추출하되, 질문 자체는 법령 본문을 확인하며 직접 작성해 그래프 구조가 정답을 유리하게 만드는 순환을 막았습니다. 결과는 가설과 반대였습니다. hit@1은 벡터 0.539, 하이브리드 0.455, 그래프 0.279로 벡터가 가장 높았고(McNemar p=0.0005), 그래프는 recall@10 0.945로 후보 회수에서만 우위였습니다. 리랭커를 켜면 세 전략 모두 hit@1 약 0.73으로 수렴합니다. 거절 정확도 1.000, 인용 정확도 0.922, 근거성 0.935도 같은 하네스로 측정했습니다. 이 평가가 그래프 검색의 역할을 "정밀도 엔진"에서 "회수 레버"로 바꿔 놓았습니다.

## 서빙과 데모

FastAPI로 서빙하고, 진행 상황은 LangGraph 스트리밍을 SSE로 내보내 검색 전략 선택과 근거 판정 과정이 화면에 보이게 했습니다. 프론트는 Next.js 정적 내보내기로, 실제 실행을 녹화한 트레이스 5건을 키 없이 재생하는 데모 모드를 기본으로 두고 Vercel에 배포했습니다.

## 배운 것

구조적으로 그럴듯한 가설도 골드셋과 통계 검정 앞에서는 뒤집힐 수 있습니다. 평가 하네스를 먼저 만들어 두면 "그래프가 좋다더라"가 아니라 "그래프는 recall 레버로 쓰고 1순위는 리랭커에 맡긴다" 같은 설계 판단을 근거와 함께 내릴 수 있었습니다. 법령처럼 참조 구조가 명시적인 도메인에서는 관계 추출에 LLM 없이 정규식만으로도 엣지 16,281개를 만들 수 있다는 점도 확인했습니다.
