법령 RAG 에이전트 — StateGraph 흐름

노드 7개, 거절 경로 2개, 재시도 루프 2개. LLM 호출은 분류·근거 판정·생성 세 지점뿐이고 라우팅은 결정적 매핑이 합니다.

classify — 질문 유형 분류 (LLM ①)
단일 조문 · 정의 · 멀티홉 · 답변 불가 → 조문 직접 지목 시 분류 생략, 정규식 파싱으로 해당 조문 고정
↓ 유형 → 전략은 dict 매핑 (single→hybrid · definition→vector · multihop→graph)
retrieve — 검색
BM25 · 벡터(Qdrant) · 그래프(Neo4j)를 RRF로 병합 · cross-encoder 리랭킹
↓
grade_evidence — 근거 충분성 판정 (LLM ②)
부족하면 질의 재작성 · 재작성 질의가 실제로 달라졌을 때만 재검색
↓ 충분
generate — 답변 생성 (LLM ③)
검색 결과 밖 조문 인용 금지 · structured output으로 파싱 실패 제거
↓
verify — 인용 검증
답변의 인용이 실제 검색 결과와 일치하는지 확인
↓
답변 + 근거 조문 인용
각 노드의 상태 변경분을 SSE로 스트리밍 → 화면에 진행 과정 노출
refuse — 즉시 거절
분류가 답변 불가로 판정하면 검색 없이 거절 (비용 절약)
↖ classify에서 분기
재검색 루프 (최대 2회)
grade_evidence 부족 판정 → 재작성 질의로 retrieve 재진입
↖ grade_evidence ⇄ retrieve
refuse_low_confidence
재검색을 다 써도 근거가 부족하면 "확신이 낮다"로 거절
↖ grade_evidence에서 분기
재생성 루프 (최대 2회)
verify 실패 → generate 재진입
↖ verify ⇄ generate
